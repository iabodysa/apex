# Apex Habitat 0.7.2

This patch release hardens reliability, performance, and security across the app, and completes the Arabic interface translations. No data migration is required.

## What changed for users

- Submitting or cancelling Utility Bills, Checkouts, Assignments, and maintenance Work Orders is now all-or-nothing — a failure no longer leaves a half-saved record or a bill without a matching ledger entry.
- The Room generator wizard now safely escapes the building code when previewing room and bed names.
- The Checkout Pending Clearance and cleaning reports load noticeably faster.
- Occupancy percentage and asset depreciation figures are now calculated live and capped correctly, so the numbers you see are accurate.
- The Lease Expiry Watchlist now respects the building and date filters you choose.
- Room updates, lease schedule regeneration, and material-template loading now enforce per-record permissions.
- The QR resident-request form now correctly saves the request type and mobile number, and limits overly long descriptions.
- The full Arabic interface is now translated, and several previously broken translation lines were repaired.

## What changed under the hood

- Background scheduler jobs roll back cleanly and keep processing if a single record fails.
- Daily cost allocation caches building data to avoid repeated lookups.
- The duplicate-workspace cleanup migration was corrected so it can no longer remove the wrong workspace copy.
- Migration patches gained safety guards (table-existence and missing-record checks) to prevent failed upgrades.
- New automated regression tests guard translation integrity, the workspace cleanup rules, and HTTP method enforcement.

## If you are updating an existing site

Run `bench migrate` and clear cache after pulling this release. On the next Desk boot, Frappe will read this change log and show the update dialog once per user.
