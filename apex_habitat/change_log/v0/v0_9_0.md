# Apex Habitat 0.9.0

The Visual & Interactive UX leap: high-speed, visual Custom Pages that replace
tedious form data entry. No change to the room generator or other core flows.

## What changed for users

- **Transfer Board** (`/app/transfer-board`) — a split-screen board (Building A vs
  Building B). Drag an occupied bed onto an empty bed (or tap source then target
  on a tablet) to move a resident; it submits a Room Bed Transfer in the
  background and repaints both panes.
- **Interactive Safety Map** (`/app/safety-map`) — a visual floor plan (rooms +
  common zones). Rooms with open maintenance pulse red/amber. Click a room or a
  floor common-zone to log a building/floor-level Safety Inspection Report.
- **Custody Kiosk** (`/app/custody-kiosk`) — a POS-style screen for storekeepers:
  pick an employee, tap large Custody Article image tiles into a cart, hit Issue.
  It submits a Custody Issue that posts to the internal store ledger.
- **Front Desk dual-image** — Quick Check-in now shows the worker's HR profile
  photo for identity verification, and both check-in and checkout can attach a
  Room Condition Snapshot (handover photo) to settle future damage disputes.
- **Native Habitat Dashboard** — a Frappe Dashboard ("Habitat Dashboard") holding
  the Beds-by-Status chart, linked from the Habitat workspace (heavy charts now
  live on a native Dashboard, keeping the Workspace a clean navigation hub).

## What changed under the hood

- All pages reuse the Front Desk contract: bulk, N+1-free read APIs and POST-only
  whitelisted writes that build+submit real documents, so native validation, the
  ledger, and locks all run — no logic is bypassed (a security-conscious design).
- Room Bed Transfer now takes a `SELECT ... FOR UPDATE` lock on the target bed so
  rapid drag-drops can't double-book it.
- New schema: `Custody Article.image`, and `room_condition_snapshot` on
  Accommodation Assignment and Checkout. Report links across every **Workspace**
  now open the report view (is_query_report). New strings are English-first via
  `ar.csv`; reads are bulk for good performance.

## If you are updating an existing site

Run `bench migrate && bench build` and clear cache after pulling. On the next Desk
boot, Frappe shows this update dialog once per user.
