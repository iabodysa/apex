# Apex Habitat 0.9.0

The visual and interactive leap: fast, tap-friendly pages that replace tedious form entry.
No change to the room generator or other core flows.

## What changed for users

- **Transfer Board** (`/app/transfer-board`) — a split-screen board (Building A vs Building
  B). Drag an occupied bed onto an empty bed (or tap source then target on a tablet) to
  move a resident; it repaints both panes the moment you're done.
- **Interactive Safety Map** (`/app/safety-map`) — a visual floor plan of rooms and common
  zones. Rooms needing attention pulse for you; tap one to log a building- or floor-level
  Safety Inspection Report on the spot.
- **Custody Kiosk** (`/app/custody-kiosk`) — a checkout-counter screen for storekeepers:
  pick an employee, tap big article-image tiles into a cart, and hit Issue.
- **Front Desk dual-image** — Quick Check-in now shows the worker's HR profile photo so
  you check in the right person, and both check-in and checkout can attach a room-condition
  snapshot to settle any future damage dispute.
- **Native Habitat Dashboard** — a Frappe Dashboard holding the Beds-by-Status chart,
  linked from the Habitat workspace, so the heavy charts live on a dashboard and keep the
  workspace a clean navigation hub.

## What changed under the hood

- All the new pages read fast and route every action through real documents, so every
  existing check runs as normal.
- New fields back the article images and the room-condition snapshots, and report links
  across the workspaces now open straight into the report view. New strings are
  English-first via `ar.csv`. Security & reliability improvements.

## If you are updating an existing site

Run `bench migrate && bench build` and clear cache after pulling. On the next Desk boot,
Frappe shows this update dialog once per user.
