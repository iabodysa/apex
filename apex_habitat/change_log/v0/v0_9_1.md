# Apex Habitat 0.9.1

This release restores guided onboarding to the workspaces and wires up the form
"Connections" tabs. No change to the room generator, maintenance, or other flows.

## What changed for users

- **Guided onboarding restored** — the navigation-first redesign had dropped the
  setup banner. It's back, now placed per workspace:
  - **Setup** → "Habitat Operations Setup" (configure settings, create site/
    building/rooms/beds, review the safety catalog, record an assignment).
  - **Accommodation** → "Accommodation Go-Live" (open the Front Desk, record an
    assignment, review the Active Resident Register).
  - **Safety** → "Safety Readiness" (review the safety catalog, log an
    inspection, open the Safety Map).
- **Form Connections** — opening an **Accommodation Building** now shows its
  linked Rooms, Beds, Assignments, Requests, Licenses, Custody, Leases, Utility
  and scheduling records grouped in the Connections tab. Accommodation Site and
  Accommodation Room get the same drill-down connections.

## What changed under the hood

- Onboarding is attached via the v15 mechanism — an `onboarding` content block in
  each workspace (the Workspace doctype has no onboarding field in v15) —
  referencing existing/new Module Onboarding fixtures with real, verified step
  targets (DocTypes, the Front Desk/Safety Map pages, and reports).
- Connections come from each DocType's native `links` array; every link fieldname
  was verified to exist and point back to the parent, so nothing renders broken.
  This is additive metadata — no controller, security, or performance impact, and
  it does not duplicate the existing Building header dashboard.

## If you are updating an existing site

Run `bench migrate` and clear cache after pulling. Ensure System Settings →
"Enable Onboarding" is on to see the setup banners. On the next Desk boot, Frappe
shows this update dialog once per user.
