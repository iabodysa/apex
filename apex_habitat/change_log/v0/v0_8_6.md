# Apex Habitat 0.8.6

This release adds the visual Front Desk check-in board and completes idle-resident
accountability. No change to the room generator, maintenance, or other flows.

## What changed for users

- **Front Desk (new page)** — a visual `building → floor → room → bed` board at
  **Front Desk** (`/app/front-desk`). Beds are colour-coded: green = available &
  ready (click to check in), red = occupied (shows the resident, click to check
  out), amber = available but the room is not ready, grey = out of service.
  - **Quick check-in** opens a short dialog (employee, project, date) and creates
    + submits a real Accommodation Assignment, so all existing validation, the
    bed lock that prevents double-booking, and occupancy recompute run normally.
  - **Quick check-out** vacates a bed in one click; if the resident still holds
    custody items it opens the full Checkout form instead.
  - **Resident Supervisors** can now check residents in and out from the board
    (granted submit on Accommodation Assignment and Accommodation Checkout).
- **Idle Resident accountability** — reporting an idle resident now automatically
  raises a follow-up task for the responsible department (HR / Operations /
  Legal) and a daily job accrues **days idle** and an **estimated accommodation
  cost bleed** (from the memo ledger — no GL), making the financial cost visible.

## What changed under the hood

- The Front Desk is a standard Frappe Page (`frappe.ui` + native dialogs, no SPA
  or external libraries); its read API returns the whole board in a few bulk
  queries (no N+1), and both write actions are POST-only whitelisted endpoints
  that route through the existing controllers — no validation is bypassed
  (a security-conscious design).
- Idle-resident routing reuses native ToDos and is idempotent; the aging job
  follows the established paginated, error-isolated scheduled-job pattern and
  posts a timeline note every seventh idle day for good visibility/performance.
- The idle notification ships disabled-by-default. New strings are English-first,
  translated through `ar.csv`. The **Workspace** layout is unchanged.

## If you are updating an existing site

Run `bench migrate` and clear cache after pulling. The Front Desk page, the new
permissions, and the disabled notification apply automatically. On the next Desk
boot, Frappe shows this update dialog once per user.
