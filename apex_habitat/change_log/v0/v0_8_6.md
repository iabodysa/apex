# Apex Habitat 0.8.6

This release lets you check residents in and out from a visual board, and never lose track
of an idle worker. No change to the room generator, maintenance, or other flows.

## What changed for users

- **Run the front desk by colour (new page)** — a visual `building → floor → room → bed`
  board at **Front Desk** (`/app/front-desk`). Beds are colour-coded: green = available
  and ready (tap to check in), red = occupied (shows the resident, tap to check out),
  amber = available but the room isn't ready, grey = out of service.
  - **Quick check-in** opens a short dialog (employee, project, date) and creates a real
    Accommodation Assignment, so every existing check runs as normal.
  - **Quick check-out** vacates a bed in one tap; if the resident still holds custody
    items it opens the full Checkout form instead.
  - **Resident Supervisors** can now check residents in and out straight from the board.
- **Never pay for a forgotten worker** — flagging an idle resident now raises a follow-up
  for the responsible team (HR, Operations or Legal), and a daily job counts the **days
  idle** and an **estimated accommodation cost** so the cost is always in plain sight.

## What changed under the hood

- The Front Desk is a standard Frappe page that returns the whole board fast and routes
  every action through the existing controllers. Idle-resident follow-ups reuse native
  to-dos and post a timeline note as the days add up.
- The idle notification ships switched off. New strings are English-first, translated
  through `ar.csv`. The workspace layout is unchanged. Security & reliability
  improvements.

## If you are updating an existing site

Run `bench migrate` and clear cache after pulling. The Front Desk page, the new
permissions and the disabled notification apply automatically. On the next Desk boot,
Frappe shows this update dialog once per user.
