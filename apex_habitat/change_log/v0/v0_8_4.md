# Apex Habitat 0.8.4

This release adds scheduled report emails, reusable email templates, follow-up
tasks, and navbar quick links. No change to the room generator or other flows.

## What changed for users

- **Scheduled report emails (Auto Email Reports)** — Supplier Cost Recovery
  (monthly), Occupancy Trend, Maintenance Backlog, and Scheduled Task Compliance
  (weekly) can now be emailed automatically. Shipped **disabled** with a
  placeholder recipient: set your real recipients and enable.
- **Reusable email templates** — pre-written bodies for Building License renewal,
  Lease renewal, and Resident Request acknowledgement, available from the Email
  action on those documents.
- **Follow-up tasks (ToDo)** — assigning a Resident Request to a user now creates
  a ToDo in that user's desk queue; resolving/closing the request closes the task
  automatically.
- **Navbar quick links** — the Help dropdown gains links to the Command Center
  and Setup workspaces.

## What changed under the hood

- Auto Email Reports and Email Templates follow the disabled/idempotent seed
  pattern, so an update never starts emailing until configured — protecting
  existing sites (a security/stability-conscious default).
- The navbar links are added by an **additive** patch that appends to the
  existing Help dropdown rather than shipping a Navbar Settings fixture (which
  would overwrite a site's own navbar on migrate).
- ToDo follow-ups reuse Frappe's native assignment, are idempotent (no duplicate
  task per assignee), and add no maintenance overhead. Report emails reuse the
  existing reports for good performance. The **Workspace** layout is unchanged.

## If you are updating an existing site

Run `bench migrate` and clear cache after pulling. The Auto Email Reports, Email
Templates, and navbar links are seeded automatically (reports disabled). On the
next Desk boot, Frappe shows this update dialog once per user.
