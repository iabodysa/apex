# Apex Habitat 0.8.3

This release adds native Frappe automation and visual paradigms (Tier 1). No
change to the room generator or other existing flows.

## What changed for users

- **Calendar views** — Scheduled Task Instances (by due date), Building Licences
  (by expiry), Subcontractor Service Orders (by scheduled date), and
  Accommodation Leases (start → end) can now be viewed on a calendar.
- **Room Readiness Kanban** — a housekeeping board to drag rooms across
  readiness columns (Needs Cleaning → Needs Repair → Ready), alongside the
  existing Resident and Maintenance request boards.
- **Auto-assignment rules** — new maintenance requests and resident requests can
  be auto-assigned to a team (load-balancing / round-robin). Shipped **disabled**
  with a placeholder user: replace it with your real team and enable.
- **Event notifications** — new email Notifications for: a maintenance request
  assigned to you, a new maintenance request awaiting triage, a resident request
  waiting for evidence, a new resident request, and a submitted custody damage
  assessment. All shipped disabled until an admin enables them.

## What changed under the hood

- Assignment Rules and Notifications follow the established disabled-by-default
  seed pattern, so an update never starts emailing or assigning until configured;
  this protects existing sites (a security/stability-conscious default).
- New notification text is English-first and translated through `ar.csv` (no
  inline Arabic). Calendar views are config-only and reuse existing fields for
  good performance.
- The **Workspace** layout is unchanged; Tier 2 workflows were intentionally
  deferred.

## If you are updating an existing site

Run `bench migrate` and clear cache after pulling. The new Kanban board,
Assignment Rules, and Notifications are seeded automatically (disabled). On the
next Desk boot, Frappe shows this update dialog once per user.
