# Apex Habitat 0.7.1

This patch release enables the official Frappe "Updated To A New Version" popup for the v0.7 upgrade and includes the user-visible v0.7 highlights below.

## What changed for users

- Workspace access is now role-restricted instead of globally visible.
- The room generator is now more flexible for real buildings, apartments, offices, storage rooms, driver rooms, supervisor rooms, and worker rooms.
- Maintenance and resident-request flows have stronger security controls.
- Scheduler warnings can now surface through Habitat Operations Alert.
- Lease payment generation and finance review flows are improved.
- Arabic translations and setup guidance were expanded for the new wizard and operational flows.

## What changed under the hood

- Bed assignment now uses safer locking to reduce double-booking risk.
- Scheduler jobs process records in batches to improve performance.
- Daily cost allocation handles leap years correctly.
- CI now checks linting, selected regression tests, migration smoke, and permissions guard rules.
- The app now ships official Frappe change-log files, so future updates can appear in the Desk update popup.

## If you are updating an existing site

Run bench migrate and clear cache after pulling this release. On the next Desk boot, Frappe will read this change log and show the update dialog once per user.
