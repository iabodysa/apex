# Apex Habitat 0.7.0

This release is the main audit-remediation sprint for Apex Habitat. It focuses on security, operational reliability, workspace clarity, and the building setup experience.

## Security and permissions

- Restricted all Habitat workspaces by role so Desk pages are no longer visible to every logged-in user.
- Converted mutating whitelisted endpoints to POST-only where required.
- Added rate limiting and honeypot protection to the public QR intake form.
- Removed unsafe permission bypasses from core write flows.
- Added permissions guard CI to protect the workspace-role baseline.

## Accommodation and room setup

- Added a more practical room generator wizard for floor-by-floor setup.
- Added support for mixed room types such as worker rooms, driver rooms, supervisor rooms, offices, storage, and maintenance rooms.
- Added floor-code based numbering to keep room and bed IDs readable.
- Added apartment/building behavior so apartment records do not force irrelevant floor fields.

## Operations, maintenance, and safety

- Added Habitat Operations Alert so scheduler warnings are visible in Desk.
- Added batch pagination for scheduler jobs to reduce timeout risk on larger data sets.
- Improved maintenance and scheduler flows with better POST guards and tests.
- Consolidated inspection finding structures toward a simpler shared model.

## Finance and reporting

- Added leap-year aware daily cost allocation.
- Added payment generation support from Accommodation Lease.
- Added and verified operational reports used by housing, maintenance, safety, and finance teams.

## Engineering quality

- Added lint, test, migration smoke, and permissions workflows in GitHub Actions.
- Migrated controller hooks toward module-level functions.
- Added regression tests for HTTP method enforcement, workspace visibility, bed booking concurrency, and schedulers.
