# Apex 1.25.0

A formal way to record accommodation safety incidents — with automatic
escalation for the serious ones — plus hardening of the Masar worker links.

## What changed for users

- **New "Habitat Safety Incident" record.** Log an accommodation safety incident
  with its date, building and specific location, an incident type (Fire,
  Electrical, Injury, Structural, Health & Hygiene, Security, Other), a
  **severity**, the number of casualties, the description, and the immediate
  action taken. Close it with resolution notes.
- **Automatic management escalation.** When an incident is submitted as **Severe**
  or **Critical**, a notification alerts management (System Manager and
  Accommodation Manager) with a link to the record — no manual chasing.
- **Severity must be chosen.** The severity field starts blank and is required,
  so an incident can never be silently filed as "Low".
- **Safer worker links.** Masar worker links are now rate-limited against abuse,
  and a link automatically stops resolving once the worker is marked **Left** or
  **Inactive** — even if the link was never explicitly revoked.

## What changed under the hood

- `Habitat Safety Incident` is a submittable DocType (Habitat module). Escalation
  is a native Frappe Notification (no hard-coded email), seeded idempotently and
  kept in sync on every migrate; it ships disabled until an admin enables it and
  email is configured.
- The five guest worker endpoints carry `frappe.rate_limiter` limits (reads
  60/min, request creation 10/hour, per IP); the limiter is a no-op outside an
  HTTP request, so tests are unaffected. `_resolve_worker` additionally rejects an
  Inactive/Left employee.

## If you are updating an existing site

- Run `bench migrate`, then `bench build` and `bench clear-cache`. To turn on
  escalation, enable the "Habitat - Severe Safety Incident" Notification and
  ensure outgoing email is configured.
