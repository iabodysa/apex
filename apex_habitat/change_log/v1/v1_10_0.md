# Apex 1.10.0

Everything about fuel now lives on one screen.

## What changed for users

- **One Fuel Request for everything.** Ordering fuel, topping up a vehicle's quota, and
  issuing, replacing or cancelling a fuel chip used to be three separate forms. They are
  now a single **Fuel Request** — choose the type at the top and only the fields that
  matter for that type appear.
- **Temporary top-ups revert on time.** When you grant a temporary quota top-up with a
  due date, it now reliably rolls back on that date, so a vehicle's quota does not quietly
  creep upward over time.
- **Less to learn, less to maintain.** One list, one set of filters and one approval path
  for all fuel activity instead of three — your existing requests are carried over
  automatically, so nothing is lost.

## What changed under the hood

- The three fuel request DocTypes were consolidated into one `Fuel Request` with a
  `request_type` selector; type-specific fields are shown conditionally, and each type
  keeps its own validation and state flow.
- Existing top-up and chip records are migrated into the unified document on update, with
  their status, submission state and history preserved.

## If you are updating an existing site

Pull and run the database migration. Existing fuel top-up and fuel chip records are moved
into the unified Fuel Request automatically; no manual data entry is required.
