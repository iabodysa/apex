# Apex 1.15.0

The shared settings now have a dedicated home in the desk.

## What changed for users

- **A dedicated Apex Core workspace.** Habitat Settings and Salis Settings — the shared
  configuration both functional modules read from — now appear in their own **Apex Core**
  workspace, with shortcuts straight to each settings page.
- **Configuration is discoverable.** The Apex Core module previously had no workspace, so its
  settings had no clear home in the desk; now there is one.
- **Scoped to the right people.** The workspace is available to the System Manager and to the
  Accommodation and Fleet managers who maintain those settings.

## What changed under the hood

- Adds a public `Apex Core` Workspace fixture for the `apex_core` module, surfacing the two
  shared Single DocTypes via shortcuts and a Settings card.

## If you are updating an existing site

Pull and run the database migration; the workspace is created automatically. Do one refresh
of the desk so the new workspace appears in the sidebar.
