# Apex 1.17.0

Apex can now serve as the backend for any frontend, with a documented integration kit.

## What changed for users

- **An integration kit.** A public integration guide (`docs/INTEGRATION.md`) plus an
  **Apex Integration Settings** page let any frontend — or any developer — install Apex as a
  backend and drive it over a clean, token-authenticated REST/RPC API. The native OAuth Client
  and Webhook tools are surfaced alongside it.
- **A tidier Apex Core workspace.** The configuration hub is reorganized into clear sections:
  Settings, Integration, Habitat Setup, Salis Setup, and Console.
- **Correct links.** What's-New and navbar items that pointed to the old, removed workspaces
  now point to the Apex Core workspace.

## What changed under the hood

- New `Apex Integration Settings` Single DocType (app-specific integration config; native
  token / OAuth Client / Webhook still do the real work). A public `docs/INTEGRATION.md`
  covering install, token auth, CORS, the API surface and an issue-based change workflow.
- The Apex Core workspace layout is rebuilt with bijective content/links; a guarded migration
  repoints stale navbar help links to `/app/apex-core`.

## If you are updating an existing site

Pull and run the database migration; the integration settings install and the navbar links are
repointed automatically. The integration guide is published in the repository under `docs/`.
