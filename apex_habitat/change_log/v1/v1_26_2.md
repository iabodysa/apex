# Apex 1.26.2

Cleanup: the old custom approval engine is fully retired now that approvals run on
native Frappe Workflow.

## What changed for users

- **No more parallel approval mechanism.** Since 1.26.0 the six Salis approval
  documents are approved through native Frappe Workflow. The old custom
  authority-tier engine and its "Approval Request" record — which nothing creates
  anymore — have been removed.
- **Tidier Salis settings.** The unused authority-tier ladder fields were removed
  from Salis Settings.
- **Same controls.** Segregation of duties (a maker cannot approve their own
  document) is unchanged — it is enforced by each workflow's approve transition.

## What changed under the hood

- Removed the tier engine from `salis_lib` (tier ladder, `ensure_approval`),
  dropped the `Approval Request` DocType and its number card, workspace links and
  permission hooks, and removed the orphaned tier fields from Salis Settings — via
  a guarded drop patch. Non-tier utilities (vehicle/driver locks, timeline notes,
  transport drive/revert) and the payment segregation-of-duties hooks were kept.
- Tests were updated: the tier-engine test was removed and the shared test helper
  relocated; the workflow tests no longer mint an Approval Request.

## If you are updating an existing site

- Run `bench migrate`; the drop patch removes the DocType automatically.
