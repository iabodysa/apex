# Apex 1.14.0

Operations alerts now clear themselves more completely and keep a resolution trail.

## What changed for users

- **Alerts close when the cause is fixed.** When the situation behind an alert is
  resolved — a licence renewed, a vehicle back in use, a fuel top-up reverted — the alert
  now closes on its own, so your alerts list reflects reality without manual cleanup.
- **Excessive top-up alerts clear too.** The one alert that used to linger — an excessive
  fuel top-up — now clears automatically once the temporary top-up is reverted or the
  vehicle's consumption is back within its quota.
- **A resolution trail.** Resolved alerts now record exactly when they cleared and the
  reason, so you can see how an issue was closed rather than guessing.
- **Less standing noise.** Your open-alerts view shows only what still needs attention.

## What changed under the hood

- `Operations Alert` gains read-only `resolved_on` and `resolution_note` fields.
- The daily reconciler now also clears `Excessive Topup` alerts (conservatively, only when
  both sources are clear), and reverting a temporary top-up resolves its alert immediately.
- A migration backfills `resolved_on` on previously resolved alerts so the trail is
  complete.

## If you are updating an existing site

Pull and run the database migration; the backfill is automatic and safe to re-run.
