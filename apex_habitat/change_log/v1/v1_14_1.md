# Apex 1.14.1

The driver portal opens reliably for signed-in users.

## What changed for users

- **The driver portal opens reliably.** Signed-in drivers and staff can now open the
  portal without it failing to load. A session-handling issue that could block the page
  from starting is fixed.
- **Refreshed project overview.** The project README now accurately describes the full
  application — all of its modules and its scheduled automation — instead of an older,
  single-module description.
- **Security and reliability improvements** under the hood.

## What changed under the hood

- The driver portal page now supplies the correct session token, so the app's requests
  are accepted instead of being rejected for signed-in users.
- Internal permission and reliability hardening across Movement documents.

## If you are updating an existing site

Pull, run the database migration, and clear cache. If the portal was open in a browser,
do one hard refresh so it picks up the new page.
