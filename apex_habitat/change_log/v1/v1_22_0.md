# Apex 1.22.0

A new portal theme, a genuinely Frappe-styled theme, and a fixed worker-route page.

## What changed for users

- **A new "Gemini" portal theme.** A premium dark, neon / glassmorphism look joins the driver-portal
  themes — pick **AFMCO**, **Frappe Standard**, **Dark**, or **Gemini** in the Salis Portal Theme settings.
- **The "Frappe Standard" theme now genuinely looks like Frappe.** It was redesigned with its own layout
  and feel (Frappe blue, system font, soft shadows, a thin utility header, denser list-style rows) — clearly
  distinct from the AFMCO theme, not just a colour swap.
- **The worker-route page (`/masar`) now works.** It was returning an error on every load; fixed. Also: the
  Apex Core workspace is restored to a complete-but-organised hub (settings up front, then Setup & Master
  Data, Admin Health, and Tools), and the newly-added desk strings are translated to Arabic.

## What changed under the hood

- New `[data-theme="gemini"]` token set + component overrides; "Gemini" added to the Salis Portal Theme
  options and the slug map. The Frappe theme gained a distinct `[data-theme="frappe"]` layout block (plus
  additive class hooks in `App.vue`); AFMCO and Dark are unchanged.
- `www/masar.html` opened with a Django-style `{% comment %}` tag, which Frappe's Jinja does not register —
  so the template failed to compile (a 500 in every state). Replaced with native `{# … #}`.
- Apex Core workspace JSON restored; `translations/ar.csv` +32 rows (Salis Portal Theme, the new number
  cards, section headers).

## If you are updating an existing site

Pull, run the database migration, rebuild assets (`yarn build` in `driver_portal` or `bench build`), and
clear the cache.
