# Apex 1.19.0

A redesigned, themeable driver portal and a fully Arabic desk.

## What changed for users

- **The driver portal has a new, mobile-first design with selectable themes.** Pick the look
  from the new **Salis Portal Theme** settings: **AFMCO** (the company brand — flat, green,
  Montserrat), **Frappe Standard**, or **Dark**. An optional accent colour and logo can be set
  too. The portal stays in **English** by design, because the driver workforce is multilingual.
- **The desk is now fully Arabic.** More than 300 desk strings that were still showing in
  English — across onboarding, reports, charts, number cards, dashboards, workspaces, and the
  Integration Settings — are now translated.
- **The Habitat workspace reads correctly in Arabic again.** Its label was being translated to a
  generic "Home" term, which made it look like a system page; it is now a clear, recognisable name.

## What changed under the hood

- New Single DocType **Salis Portal Theme** (`theme`, `show_brand`, `accent_color`, `brand_logo`)
  with an idempotent seed (defaults to AFMCO). `www/driver.py` resolves the active theme and
  passes it to the page as a `data-theme` attribute consumed by a CSS design-token system, so all
  three themes are pure token swaps with no hard-coded colours. The Vue portal was rebuilt; the
  CSRF flow and every portal API call are unchanged.
- `translations/ar.csv` gained 302 app-specific rows, each verified against Frappe's merged Arabic
  map so none duplicates a string the framework already ships. The `Habitat` label was corrected.

## If you are updating an existing site

Pull, run the database migration (it creates the Salis Portal Theme settings, seeds the AFMCO
default, and re-imports the translations), then rebuild assets and clear the cache.
