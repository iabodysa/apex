# Apex 1.13.0

Dispatch trips now run on a workflow — completing the Movement approval suite.

## What changed for users

- **A clear path for every trip.** A dispatch trip now moves through **Planned →
  Dispatched → Completed**, with a status you can read at a glance.
- **Completing a trip does the right things automatically.** Marking a trip Completed
  fulfils its linked transport request and advances the vehicle's odometer in one step —
  no separate manual updates.
- **Calling off a completed trip is clean.** Cancelling a completed trip reverses those
  effects: the transport request returns to Scheduled and the trip's posting is undone.
- **The whole Movement suite is now guided.** Transport requests, rental settlements,
  driver clearances, payments, support tickets, sponsorship transfers, fuel requests,
  claims and exceptions — and now dispatch trips — all change status only through guided,
  permitted actions by the right role.

## What changed under the hood

- `Dispatch Trip` adopts a native Frappe Workflow (the tenth and final status document on
  the Salis approval spine), reusing its `status` field as the workflow state. Completion
  is the submit step, so the existing fulfilment, odometer and ledger side-effects fire
  exactly as before; cancellation reverses them.
- The hand-rolled transition checks are retired in favour of the workflow; readiness,
  completion-note and compliance validations are preserved.

## If you are updating an existing site

Pull and run the database migration; the Dispatch Trip workflow is seeded automatically.
Existing trips keep their current status and continue from there.
