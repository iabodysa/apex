# Apex 1.16.0

All configuration and admin tools now share one home.

## What changed for users

- **One Apex Core workspace for everything admin.** The shared settings, the Habitat setup
  masters, the Habitat console (engines, ledgers, logs and alert counts), and the Salis setup
  masters are now gathered into a single **Apex Core** workspace.
- **Less sidebar clutter.** The three separate workspaces — Habitat **Setup**, Habitat
  **Console**, and **Salis Setup** — are removed; their links, shortcuts and cards are all
  carried over into Apex Core, organized into clear sections.
- **One place to configure both modules.** Habitat and Salis configuration now live
  side by side, so there is a single, predictable home for administration.

## What changed under the hood

- The three workspaces are merged into the `Apex Core` workspace fixture (39 links across
  labelled card sections) and removed; a guarded, idempotent migration deletes the old
  Workspace records, and the Salis sidebar link is repointed to Apex Core.

## If you are updating an existing site

Pull and run the database migration; the old workspaces are removed and Apex Core carries
their content automatically. Refresh the desk once so the sidebar updates.
