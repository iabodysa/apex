# Apex 1.27.0

Onboarding an arriving worker is now a single screen — hotel reception style.

## What changed for users

- **New Arrivals Desk** (`/app/arrivals-desk`). Pick an arriving worker, see their
  identity card (photo, project, current bed, custody and Masar-link status), and
  complete the whole intake from one place:
  1. **House** them (assign a building / room / bed),
  2. **Issue custody** items,
  3. **Generate their Masar link + QR** and share it over WhatsApp,
  4. **Request their transport**.
- **Fail-soft permissions.** Each action reuses the existing flow; any action you
  are not allowed to perform is shown disabled with a reason, so the page never
  exposes a new permission path.

## What changed under the hood

- A standard desk Page (`arrivals_desk`) plus a thin read-only
  `get_arrival_card(employee)` reader. Every action calls an existing whitelisted
  endpoint (front-desk check-in, custody-kiosk issue, Masar token issue, and a
  prefilled Transport Request) — no new business logic. Arabic desk strings added.

## If you are updating an existing site

- Run `bench migrate`, `bench build` and `bench clear-cache`. Open it at
  `/app/arrivals-desk` (a workspace shortcut can be added later).
