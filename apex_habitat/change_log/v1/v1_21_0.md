# Apex 1.21.0

A clean settings hub, at-a-glance KPI tiles on every Salis workspace, and a training guide.

## What changed for users

- **The Apex Core workspace is now a focused settings hub.** It shows the four settings up
  front — Habitat Settings, Salis Settings, Apex Integration Settings, and **Salis Portal Theme**
  (where you switch the driver-portal theme) — plus a small Roles & Access and an Admin Health row.
  The master-data link clutter and operational consoles were removed (master data lives in the
  module workspaces where it belongs).
- **Every Salis workspace now shows KPI number cards.** Fleet, Fuel, Driver Compliance, Rentals &
  Costs, Representatives & Fleet, and Workers Transport each carry at-a-glance tiles — open requests,
  active drivers and vehicles, licenses expiring soon, settlements pending, litres dispensed, and more.
- **A training guide and a refreshed README.** A per-DocType permission & usage training guide
  (`docs/TRAINING.md`) and an updated README describing the themed portal and fully-Arabic desk.

## What changed under the hood

- New `is_standard` Number Cards across the six Salis workspaces, each filter validated against its
  DocType's fields. Apex Core workspace rewritten as a settings-only hub.
- Removed dead code: the deprecated **Habitat Operations Alert** DocType (superseded by Operations
  Alert; the Unresolved Alerts card is re-pointed) + dead Select options + the empty `habitat_city`
  DocType, all behind guarded, idempotent cleanup patches.
- **Masar Phase 1b:** a read-only "my worker route today" page (`/masar`) + identity-scoped endpoints
  + test. A rich, bench-only demo generator (local, not published). frappe-master skill note corrected.

## If you are updating an existing site

Pull, run the database migration (it loads the new number cards, runs the cleanup patches, and
refreshes the Apex Core workspace), then rebuild assets / clear the cache.
