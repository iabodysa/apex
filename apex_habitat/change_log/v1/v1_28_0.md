# Apex 1.28.0

Driver support tickets now run on ERPNext's native Issue desk — no more bespoke
ticket DocType.

## What changed for users

- **Support tickets are now native Issues.** A driver raises and follows up on a
  ticket from the portal exactly as before, but each ticket is a standard ERPNext
  **Issue** — with response/resolution **SLA** targets, a native communication
  timeline, and the standard Issue tooling on the desk.
- **Right people, right access.** A driver sees only the Issues they raised; fleet
  managers/supervisors/project managers can read and work them; finance and audit
  can read. The driver portal is unchanged.
- **Categories and priorities preserved.** Vehicle / Fuel / Attendance / Salary /
  Other become Issue Types; Low / Medium / High / Urgent become Issue Priorities.

## What changed under the hood

- Removed the custom Support Ticket + Ticket Message DocTypes and their workflow
  (a guarded drop patch). The portal endpoints (`raise_support_ticket` /
  `my_support_tickets`) now read/write Issue, scoped by a new `Issue.custom_driver`
  field (shipped via a `sync_customizations` file). An idempotent seed creates the
  Issue Types, Priorities, one default Service Level Agreement, and the Salis role
  permissions on Issue (per-step savepoints so one optional step can't undo the
  others). Kanban, assignment-rule, dashboard, workspace and connection references
  were repointed to Issue.

## If you are updating an existing site

- Run `bench migrate`, then `bench build` and `bench clear-cache`. (If the site has
  no Holiday List, the SLA is skipped until one exists — Issue still works.)
