# Apex 1.26.1

A scope cleanup: the app drops a government-relations record that did not belong
in its accommodation-cost and fleet-operations core.

## What changed for users

- **Removed "Sponsorship Transfer Case".** This document tracked the bureaucratic
  kafala (sponsorship) transfer process with a Qiwa status — a government-relations
  procedure, not a cost or operations record.
- **No data impact.** It carried only test data in this pre-production app; the
  document, its approval workflow and its dashboard card are gone.
- **Cleaner Compliance workspace.** The Drivers / Compliance workspace no longer
  shows the sponsorship-transfer links and card.

## What changed under the hood

- The DocType, its native Workflow, its number card, its permission hooks
  (`permission_query_conditions` / `has_permission`), the employee-links entry and
  the workspace references were all removed. A guarded, idempotent patch drops the
  DocType and its table on existing sites (a no-op on fresh installs).
- If cost attribution per labour supplier is wanted later, it should be modelled as
  a cost *dimension* (a supplier link feeding the Cost by Dimension report), not as
  a transfer-process document.

## If you are updating an existing site

- Run `bench migrate`; the drop patch removes the DocType automatically.
