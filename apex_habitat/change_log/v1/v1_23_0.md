# Apex 1.23.0

Masar grows from a single read-only page into a complete self-service app for the
field worker — the housed-and-transported employee.

## What changed for users

- **Masar is now the worker's app.** Opening `/masar` with a worker's personal link
  shows a mobile app with four areas: **My Profile** (details and documents, with
  iqama / passport expiry warnings), **My Accommodation** (their building, room and
  bed, plus the in-charge contact to call or message), **My Transport** (their
  pickup point and time, route stops, and the assigned vehicle and driver), and
  **My Requests** (raise a maintenance / transport / HR request and follow its
  status). It defaults to Arabic and can switch to English.
- **Workers sign in with a personal link, not an account.** Each worker has an
  unguessable personal link (printable as a QR from the new *Masar Worker Token*
  record). No username or password to manage in the field.
- **The driver "my route today" view moved to the driver portal.** It now lives in
  `/driver` as a **My Route** screen — where the driver already works — instead of
  on the worker's page.
- **New "Cost by Dimension" report.** Accommodation cost broken down by company,
  building and project, with a total row.
- **Fuel Approval Console tidy-up.** The metric tiles no longer show stray markup
  next to the numbers.

## What changed under the hood

- New `Masar Worker Token` DocType issues and scopes per-worker access; all worker
  endpoints are guest-accessible but strictly token-scoped server-side — a worker
  can only ever see their own data, and no employee id is ever trusted from the
  client.
- New `worker_portal` Vue app (mirrors the driver portal's theming, tokens and
  build), served by a repurposed `www/masar` shell. Requests reuse the existing
  *Accommodation Resident Request* model — no new ticketing engine.
- `City.country` is now a real `Link → Country` instead of free text.

## If you are updating an existing site

- Run `bench migrate` (creates the *Masar Worker Token* DocType), then
  `bench build` and `bench clear-cache` to pick up the new worker portal assets.
- Generate a worker's link from their *Masar Worker Token* record (use **Show Link
  & QR**), then share that link / QR with the worker.
