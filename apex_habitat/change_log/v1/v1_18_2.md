# Apex 1.18.2

Vehicle, driver and passenger-manifest lists are now project-scoped.

## What changed for users

- **Lists respect your projects.** A supervisor scoped to a project now sees only that
  project's **vehicles**, **drivers** and **passenger manifests** in the desk lists, instead
  of every project's — closing a list-enumeration gap.
- **Your own profile still shows.** A driver continues to see their own driver record, so the
  self-service view is unaffected.
- **Oversight is unchanged.** Fleet Managers, Finance and auditors still see everything.

## What changed under the hood

- Wired the project-scoping query conditions for `Salis Vehicle`, `Salis Driver` and
  `Passenger Manifest` (the Driver query keeps the `if_owner` self-profile by OR-ing the
  owner), plus a per-document check on `Salis Vehicle`, with scoping tests covering the
  scoped, oversight and driver-self cases.

## If you are updating an existing site

Pull and run the database migration. No data changes; the scoping applies immediately.
