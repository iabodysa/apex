# Apex 1.18.3

Leaner Arabic translations.

## What changed for users

- **Standard terms use Frappe's own translations.** About 60 generic Frappe/ERPNext terms
  (statuses, month names, generic field labels) were removed from the app's Arabic file, so
  they inherit Frappe's shipped translations instead of duplicating them.
- **Domain strings are untouched.** Every Salis and Habitat domain term stays translated in
  the app's file.
- **No regressions.** The few standard terms Frappe does **not** translate (for example
  "Approved") were kept, so no label falls back to English.

## What changed under the hood

- Trimmed `apex_habitat/translations/ar.csv` to the app-specific strings, after verifying each
  removed term is actually shipped by Frappe or ERPNext.

## If you are updating an existing site

Pull, run the database migration, and rebuild translations (`bench build` / clear cache).
