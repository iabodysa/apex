# Apex 1.18.4

Driver portal check-ins now count as attendance, and the fuel ledger upgrades
cleanly on older sites.

## What changed for users

- **A portal check-in now records attendance.** When a driver checks in from the
  portal, the attendance is submitted immediately, so the missing-attendance
  watcher and the supervisor-delay reconciler both recognise it. An open
  "missing attendance" alert now auto-resolves the moment the driver checks in.
- **Check-out updates the same record.** Checking out updates the already-submitted
  attendance in place (worked hours, check-out time, photos) instead of leaving a
  separate draft behind.
- **The fuel ledger upgrades safely on existing sites.** A new migration removes any
  pre-existing duplicate fuel-ledger rows before the database's unique-source guard
  is applied, so updating an older site can no longer fail midway through.

## What changed under the hood

- `driver_check_in` / `driver_check_out` persist through a shared helper that inserts
  and submits a new same-day record, or updates the submitted one in place. The
  Driver role gains `submit`, and the check-out fields are marked `allow_on_submit`.
- New pre-model-sync patch `dedupe_fuel_ledger_and_unique_index` collapses duplicate
  `(source_type, source_name)` ledger rows to the earliest and ensures the
  `unique_fcl_source` index the fuel engine relies on as its idempotency backstop.

## If you are updating an existing site

Pull and run the database migration. The de-duplication patch runs automatically and
is a no-op on a fresh install.
