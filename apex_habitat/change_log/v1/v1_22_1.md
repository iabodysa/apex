# Apex 1.22.1

The custom desk pages are more robust, more dynamic, and more comfortable.

## What changed for users

- **The desk pages no longer fail silently.** The six operational pages — Front Desk, Custody Kiosk,
  Safety Map, Transfer Board, Fuel Approval Console, and Dispatch Board — now show clear **loading**,
  **empty**, and **error (with Retry)** states instead of a blank or stale screen when a load fails.
- **Real bugs fixed.** Across the pages: silent failures with no feedback, a stale-response race (an
  older request overwriting a newer one after you switch context), a building-field clear that left
  stale data on screen, and double-submit gaps on action buttons were all fixed.
- **Smoother, more dynamic interactions.** Loading skeletons while data fetches, debounced search,
  double-submit guards, and accessibility touches (aria-labels, reduced-motion support) make the
  pages feel livelier.

## What changed under the hood

- A per-page audit (one agent per page) found ~24 genuine page-side bugs and applied ~29 UX upgrades.
  Every page's backend endpoints were cross-checked against the DocType schemas — **all clean** (no
  field / method / permission issues), so no backend changes were needed. Each page's JS and JSON were
  validated (`node --check` + JSON parse).
