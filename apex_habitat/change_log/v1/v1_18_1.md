# Apex 1.18.1

A payment request can only be raised on an approved rental settlement.

## What changed for users

- **Payments follow approval.** A rental payment request can now only be raised on an
  **Approved** settlement — no longer on a **Disputed** (contested) or not-yet-approved one.
- **Finance protection.** A contested settlement cannot be turned into a payment until it is
  resolved and approved, closing a gap where a payment could be started on a disputed amount.
- **No workflow change.** The normal path — reconcile, approve, then raise the payment — is
  unaffected; only the out-of-order case is now blocked.

## What changed under the hood

- `create_payment_request` on Rental Settlement now guards on `status == "Approved"` (in
  addition to the existing submitted check), with a regression test covering both the allowed
  and the blocked case.

## If you are updating an existing site

Pull and run the database migration. No data changes; the new guard applies immediately.
