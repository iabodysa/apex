# Apex 1.19.1

Arabic module names, a tidier settings hub, and a cleaner driver portal.

## What changed for users

- **The modules read with proper Arabic names on the desk.** Apex → أبيكس, Salis → سلس
  (it means "smooth/seamless" — fitting for the movement module), and Habitat → الإسكان
  (it previously rendered as a generic "Home" term, which made the workspace hard to find).
- **The Apex Core settings hub is tidier.** It is reorganised into clear sections — Settings,
  Roles & Access, Setup & Master Data, Admin Health, and Tools — and the **driver-portal theme
  settings are now surfaced there**, so they are easy to find.
- **The driver portal no longer leaves stray records behind.** The check-in / check-out, fuel,
  and ticket actions no longer force an early database commit; the request commits normally on
  success. This stops the portal from leaving orphaned rows on a developer/test site.

## What changed under the hood

- Removed four redundant `frappe.db.commit()` calls from the `driver_portal.py` write endpoints —
  a whitelisted request already commits on success, and the explicit commits broke test rollback
  (leaking fixtures onto dev sites). Converted the portal test classes to `FrappeTestCase` and
  added a feature-flag "portal disabled" gating test (every action blocks when the portal is off).
- `translations/ar.csv`: added Apex/Salis/Habitat module names and swept the Salis term to سلس.
- Reorganised the Apex Core workspace; surfaced the `Salis Portal Theme` settings.

## If you are updating an existing site

Pull, run the database migration, and rebuild assets / clear the cache.
