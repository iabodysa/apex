# Apex 1.20.0

The driver portal gains a language switch and personal profile/vehicle pages.

## What changed for users

- **Each driver can switch the portal language between English and Arabic.** A new
  **EN | ع** toggle in the portal header renders the whole portal in the driver's
  language, with full right-to-left layout for Arabic. English stays the default, and
  the choice is remembered on the device — so a multilingual driver workforce each gets
  a portal they can read.
- **A driver profile page.** A driver can now open their own profile — full name, status,
  phone, project, employee, license number and expiry (with the expiry colour warning),
  and their current vehicle.
- **A vehicle detail page.** A driver can see their assigned vehicle — plate, category,
  status, ownership, and the assignment start date — with a clear empty state when no
  vehicle is assigned.

## What changed under the hood

- New read-only, identity-scoped endpoints `get_driver_profile` and `get_my_vehicle` in
  `driver_portal.py` — they resolve the driver from the session (never client-supplied),
  return only the caller's own data, and never commit. New Vue i18n layer (`en`/`ar`
  dictionaries + RTL), `Profile` and `Vehicle` pages, a sixth profile tab; the portal was
  rebuilt.
- Test hardening: re-added the `DuplicateEntryError` guards to the shared portal test
  helpers (the full-suite `exists()`-vs-DB cache skew), keeping the `FrappeTestCase`
  isolation and commit-free endpoints introduced in 1.19.1.

## If you are updating an existing site

Pull, run the database migration, and rebuild assets / clear the cache.
