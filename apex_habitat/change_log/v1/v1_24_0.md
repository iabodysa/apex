# Apex 1.24.0

Issuing a worker their personal Masar link is now a one-click job for the people
who actually onboard them — with a QR that works and a WhatsApp share.

## What changed for users

- **Issue a worker's link from where you already are.** The Accommodation
  Assignment form has a new **Issue Masar Link** button: open the worker's
  assignment, click it, and get their personal link, a scannable QR, and a share
  action — no need to open a separate record.
- **The QR code now renders.** Previously the QR could fail and only the link
  showed; it now produces a reliable, scannable code.
- **Share over WhatsApp from the browser.** A **Send via WhatsApp** button opens
  WhatsApp to the worker's registered number with the link pre-filled — no
  WhatsApp integration or setup required.
- **Who can issue a link.** Accommodation supervisors, HR, and fleet supervisors
  (and System Manager) can issue worker links.

## What changed under the hood

- The QR is rendered with `pyqrcode` (bundled with Frappe) as an inline SVG, so
  it has no extra dependency and never blocks the dialog.
- WhatsApp uses a plain `wa.me` deep link; the worker's number is normalised to
  international digits client-side, and the button hides when no usable number is
  on file.
- The `Masar Worker Token` DocType moved from the Salis module into **Apex Core**
  as shared worker-access infrastructure (a guarded patch re-homes it; the worker
  API stays in Salis). Its permissions now list the issuing roles above.

## If you are updating an existing site

- Run `bench migrate`, then `bench build` and `bench clear-cache`. The DocType
  move and permissions apply automatically.
