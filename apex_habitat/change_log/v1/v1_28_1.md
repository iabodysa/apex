# Apex 1.28.1

A correctness fix for the support-ticket SLA.

## What changed for users

- **SLA uses the right calendar.** The support-ticket Service Level Agreement now
  picks the default Company's holiday list — the real business calendar — so the
  response/resolution timers pause on the correct holidays.
- **Safer setup.** If no company holiday list is set, it falls back to any holiday
  list, and if none exists the SLA is simply skipped (Issue still works) until an
  admin configures one.

## What changed under the hood

- `issue_seed._pick_holiday_list` now prefers `Company.default_holiday_list`, then
  the HR Settings default (read best-effort, as its field name varies by HRMS
  version), then any list. Combined with the per-step seed savepoints, a missing
  or odd holiday list can never block the Issue Types / Priorities / permissions
  from seeding.

## If you are updating an existing site

- Run `bench migrate`. An SLA already created keeps its holiday list; set it to the
  company calendar manually if it was seeded earlier from a non-business list.
