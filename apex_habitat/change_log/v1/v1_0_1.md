# Apex 1.0.1

A focused fix for the Driver Portal at **/driver**.

## What changed for users

- **Driver Portal opens cleanly for everyone.** Previously, opening `/driver` while
  signed in as a user who is not linked to a driver profile (for example an
  administrator previewing the page, a manager, or a driver whose Employee record
  has not been linked yet) returned a server error in the background and logged an
  uncaught error in the browser. The portal now recognises that state up front and
  shows the friendly message "Your account is not linked to a driver profile."
  with no error.

## What changed under the hood

- The portal bootstrap endpoint `get_driver_context` no longer raises a permission
  error for an unlinked user. It resolves the driver softly and returns a structured
  `{ "enabled": ..., "linked": false, "driver": null }` response so the single-page
  app can render the correct screen without an HTTP 403 or an uncaught client error.
- Action endpoints (check-in/out, fuel request, support ticket, trips) are unchanged
  and still require — and are scoped to — a server-resolved driver.
- The portal SPA was rebuilt to render the "not linked" screen from the new flag.

## If you are updating an existing site

Pull, then `bench build` (or rebuild the Driver Portal assets) and clear cache. No
schema change and no migration are required.
